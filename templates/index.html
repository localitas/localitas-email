Email Filters

Filter Syntax

Filters use a one-liner syntax: conditions => actions

Conditions

from:value Sender contains value
to:value Recipient contains value
subject:value Subject contains value
body:value Body contains value
has:attachment Has attachments
is:unread Email is unread

Match Modifiers

field:value Contains (default)
field:=value Exact match
field:^value Starts with
field:$value Ends with

Boolean Logic

a b AND — both must match (space-separated)
a OR b OR — either must match
-a NOT — must not match
(a OR b) c Grouping with parentheses

Actions

archive Archive the email
delete Delete the email
star Star the email
mark read Mark as read
mark unread Mark as unread
move:folder Move to folder
label:name Apply label
forward:email Forward to email

Examples

from:newsletter => archive, mark read
from:$@github.com subject:^[CI] => archive
(from:boss OR from:cto) subject:urgent => star
has:attachment -from:=spam@junk.com => label:files
from:$@company.com => move:work
(from:newsletter OR from:digest) -subject:important => archive, mark read
